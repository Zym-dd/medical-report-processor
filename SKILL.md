# 体检报告处理器

处理体检报告（PDF 格式），提取可量化的指标，生成标准化的映射文件和数据文件。

## 使用场景

当用户出现以下情况时，调用此技能：
- 提供 PDF 体检/健康检查报告需要处理
- 提到"体检报告"、"处理报告"、"提取指标"、"生成映射"、"生成数据"
- 拖拽/附加 PDF 文件，或提供包含体检报告的文件/文件夹路径

## 输出模式 — 关键词检测

解析用户提示词以确定输出模式。关键词为中文，不区分大小写：

| 模式 | 关键词 |
|------|----------|
| **仅映射** | `映射` `mapping` `映射文件` |
| **仅数据** | `数据` `data` `数据文件` |
| **两者均输出** | `全部` `两种` `both` `all` `全部文件` `映射和数据` `数据和映射` |

未检测到任何关键词时，默认输出**两者均输出**。

## 工作流程

### 步骤 1 — 收集 PDF 文件

接受以下任意形式的输入：
- 单个 PDF 文件路径：`D:\\reports\\张三.pdf`
- 多个 PDF 文件路径：`D:\\reports\\张三.pdf D:\\reports\\李四.pdf`
- 包含 PDF 的文件夹路径：`D:\\reports\\`（递归扫描 `\*.pdf`）
- 拖拽：从用户消息中提取文件路径

将所有 PDF 收集为一个有序列表，处理顺序 = 文件在输入中出现的顺序。

### 步骤 2 — 逐份读取 PDF

按顺序读取每份 PDF，使用 Read 工具提取文本内容。体检报告可能包含内嵌文字或扫描图片。

### 步骤 3 — 检测医院名称

扫描 PDF 文本（特别是首页页眉/标题区域）中的医院名称。常见模式：
- 页眉文本如"XX医院体检报告"、"XX医院健康管理中心"、"XX体检中心"
- 查找中文医院名称模式：`\[\\u4e00-\\u9fa5]{2,}(医院|体检中心|健康管理中心|卫生服务中心|医疗中心)`
- 若存在歧义，采用第 1 页中最醒目的机构名称

### 步骤 4 — 提取患者信息与指标

对每份 PDF，**仅从 52 项模板**（参见 `assets/mappings\_template.csv`）中提取以下内容：

**患者信息**（映射到 `age`、`ID`、`sample\_name`、`sex`、`exam\_year`）：
- 姓名（姓名/受检者/被检人）
- 性别 — 统一为 `男` 或 `女`
- 年龄
- 体检年份（体检日期/检查日期）— 仅提取年份
- ID（档案号/登记号/身份证号）

**指标提取规则：**
1. 仅提取 `canonical\_name` 存在于模板（`assets/mappings\_template.csv`）中的项目
2. 不在模板中的项目 → **静默丢弃**
3. 每项匹配的记录格式：`{original\_name: "PDF 中医院使用的名称", value: "测量值", unit: "单位"}`

**智能解析 — 运用医学常识：**

血压：
- `101/59` 或 `59/101` → 收缩压 = 较大数（101），舒张压 = 较小数（59）
- `120/80 mmHg` → 收缩压=120，舒张压=80
- 若 PDF 中"血压"为单个标签，需拆分为 systolic_bp 和 diastolic_bp

带单位的值：
- `5.2 mmol/L` → 提取 `5.2`，记录单位为 `mmol/L`
- `126/89 mmHg` → 参见上述血压规则

参考范围过滤：
- PDF 中常出现参考范围如 `3.5--9.5`、`≤1.7`、`40--75`、`0-14`
- **绝不**将参考范围当作患者数据值
- 实际测量值通常位于单位之前或指标名称之后
- 若某个值与参考范围模式相似，需确认其为测量值而非参考范围

中文体检报告中常见的排版模式（多策略提取）：
- 表格格式：`项目名称 | 结果 | 单位 | 参考范围`
- 括号格式：`【甘油三酯】1.74，偏高，(≤1.7)mmol/L`
- 冒号格式：`心率：64 次/分`
- 行内格式：`丙氨酸氨基转移酶 7--40 U/L 12.1`
- 断行名称（预处理阶段）：`丙氨酸氨基转移\\n12.80\\n酶` → 拼接为 `丙氨酸氨基转移酶 12.80`
- 小结格式（第二遍）：`小结：红细胞分布宽度(SD): 48.40 fL`
- 段落冒号带单位（第二遍）：`丙氨酸氨基转移酶：76.10U/L`

首遍提取完成后，第二遍使用宽松模式扫描原文（段落内冒号、小结行、名称-参考范围-单位-值）。第三遍预处理文本拼接断行指标名称，再对拼接后的文本重新扫描。每遍只填补前一遍留下的空位 — 绝不覆盖已有值。这样可以捕获嵌入叙述文字、小结段落以及断行格式中的值，而这些是结构化表格解析器会遗漏的。

医学名称映射：
- 以 `references/medical\_synonyms.md` 为主要参考，进行医院名称→标准名称的映射
- 将 AI 推理（医学知识）作为不熟悉医院命名习惯时的后备方案
- 知识库可扩展 — 发现新模式时添加新的同义词条目
- 切勿仅依赖知识库；灵活运用医学推理

### 步骤 5 — 构建内部映射

为在所有 PDF 中检测到的每家医院构建一个映射表：

```

hospital\_name → canonical\_name → official\_feature\_name

```

映射始终从模板（`assets/mappings\_template.csv`）出发，该模板定义了 52 个项目，列名为：
`hospital\_name, canonical\_name, official\_feature\_name, official\_unit, official\_reference, official\_group\_zh`

对模板中的每一行：
1. 尝试从该医院的 PDF 中找到匹配的指标
2. 使用医学同义词 + AI 推理，按 `canonical\_name`（中文）进行匹配
3. 若医院名称 ≠ canonical_name → 在 `hospital\_name` 列填入 PDF 中医院使用的确切名称
4. 若医院名称 = canonical_name → `hospital\_name` 列留空
5. 若该医院的**任何**报告中均未找到该项 → `hospital\_name` 列留空

**按医院构建映射，而非按患者。** 来自同一医院的所有患者共用一份映射文件。

**始终在内部计算映射**，即使用户只要求输出数据文件。映射决定了提取哪些指标以及如何标注它们。

### 步骤 6 — 生成输出文件

**输出目录**：将所有输出文件放置在**第一份 PDF 所在的父目录**中。

#### 6a — 映射文件（模式为 `mapping` 或 `both` 时）

每家检测到的医院生成一份文件：

- **文件名**：`{医院名称}mappings.csv`（例如 `南京鼓楼医院mappings.csv`）
- **名称冲突**：若文件已存在 → `{医院名称}mappings2.csv` → 仍存在 → `{医院名称}mappings3.csv` → 以此类推
  - 原始（无冲突的）文件**不加**后缀
  - 第一个重复文件加后缀 `2`，第二个加 `3`，绝不从 `1` 开始

- **格式**：6 列，UTF-8 带 BOM
  1. `hospital\_name` — 该项的医院名称（若与标准名相同或未找到则留空）
  2. `canonical\_name` — 标准中文名称（来自模板）
  3. `official\_feature\_name` — 标准英文名称（来自模板）
  4. `official\_unit` — 单位（来自模板）
  5. `official\_reference` — 参考范围（来自模板）
  6. `official\_group\_zh` — 分组中文名（来自模板）

- **顺序**：严格遵循 `assets/mappings\_template.csv` 中的行顺序。禁止按类别或字母排序。

#### 6b — 数据文件（模式为 `data` 或 `both` 时）

整批数据生成一份文件：

- **文件名**：`YY,MM,DD HH:MM\[n].csv`，其中：
  - `YY,MM,DD` = 当前系统日期（年份取后两位）
  - `HH:MM` = 当前系统时间（24 小时制）
  - `n` = 本批处理的 PDF 报告数量
  - 示例：`26,07,21 09:35\[3].csv` 表示 2026 年 7 月 21 日 09:35 处理了 3 份报告

- **名称冲突**：与映射文件规则相同 — 第一个重复文件加后缀 `2`，以此类推

- **格式**：UTF-8 带 BOM
  - **第 1 行（表头）**：按模板的**严格顺序**排列的 `official\_feature\_name` 值（逗号分隔，无引号）
  - **第 2 行**：患者 1 的数据（逗号分隔，仅数值不含单位）
  - **第 3 行**：患者 2 的数据，以此类推
  - 患者顺序 = PDF 输入顺序

- **缺失值**：若某患者缺少某项指标数据，该单元格留空（连续逗号）

- **患者信息提取**：
  - `age` → 报告中的年龄数值
  - `ID` → 报告中的档案号/登记号/身份证号
  - `sample\_name` → 患者中文姓名
  - `sex` → `男` 或 `女`
  - `exam\_year` → 四位年份
  - `bmi` → 报告中的直接数值，若未提供则留空

## 重要约束

1. **模板为唯一标准**：所有 52 项及其顺序均来自 `assets/mappings\_template.csv`。禁止增加、删除或重排项目。

2. **映射文件 = 固定模板**：无论具体医院的报告内容如何，映射文件始终有固定的 52 行（加表头），顺序不变。

3. **数据文件 = 固定列**：数据文件表头始终有固定的 52 列，顺序不变。

4. **每家医院一份映射，每批一份数据**：同家医院多名患者 → 1 份映射文件。多家医院 → 多份映射文件。所有患者 → 1 份数据文件。

5. **不复用旧文件**：始终从头重新计算映射。即使输出路径中存在之前生成的映射文件，也禁止读取或复用。

6. **医学推理优先**：在将医院项目名称匹配到模板 canonical_name 时，首先使用医学知识（通过 AI 推理），其次回退到同义词知识库。这确保了对不熟悉报告格式的灵活性。

## 错误处理

- 若 PDF 无法读取或不含文字 → 跳过，记录跳过信息，继续处理剩余文件
- 若无法检测到任何医院名称 → 使用 "Unknown_Hospital" 作为后备
- 若 PDF 中无患者姓名 → 使用 `Unknown` 作为 sample_name
- 若写入 CSV 时出现编码问题 → 始终以 UTF-8 带 BOM 格式写入

## 扩展知识库

添加新的医学同义词时，编辑 `references/medical\_synonyms.md`。格式如下：

```

\## 类别名称

\- 医院名称\_1, 医院名称\_2 → canonical\_name

```

从真实报告中发现的模式随时添加条目，按与模板相同的分组进行分类管理。
